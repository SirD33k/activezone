================================================================================
                    TROUBLESHOOTING
================================================================================

Problem: "npm: command not found" or "node: command not found"
Solution:
   - Node.js is not installed
   - Download from: https://nodejs.org
   - Install LTS version
   - Restart PowerShell after installation

Problem: "Port 3001 already in use"
Solution:
   - Another process is using port 3001
   - Stop the other server.js instance
   - OR change PORT in .env file to 3002

Problem: "Port 5173 already in use"
Solution:
   - Close other Vite dev servers
   - OR the port will auto-increment to 5174

Problem: Products don't load in store
Solution:
   - Check backend server is running (http://localhost:3001)
   - Check browser console (F12) for errors
   - Verify Gym Master API is accessible
   - Test: http://localhost:3001/api/products

Problem: "Cannot connect to database"
Solution:
   - If you don't have MySQL installed, that's OK!
   - Backend will work without database (orders saved to file)
   - For database testing, install MySQL/MariaDB
   - Import: database/schema.sql

Problem: Changes not showing in browser
Solution:
   - Hard refresh: Ctrl+F5
   - Clear browser cache
   - Check file is saved
   - Verify dev server is still running

Problem: CORS errors in browser console
Solution:
   - Make sure backend server is running
   - Check .env file has correct APP_URL
   - Restart backend server after .env changes

Problem: Payment fails with test card
Solution:
   - Use exact test card: 4084084084084081
   - CVV: 408, PIN: 0000, OTP: 123456
   - Verify PAYSTACK_SECRET_KEY is test key (sk_test_...)
   - Check Paystack dashboard for errors

Problem: Google Authenticator TOTP not working
Solution:
   - Visit: http://localhost:3001/api/totp/setup
   - Scan NEW QR code
   - Update TOTP_SECRET in .env file
   - Restart backend server
   - Use fresh code from Google Authenticator (codes expire every 30 seconds)

Problem: Email notifications not sending
Solution:
   - Check BREVO_API_KEY in .env
   - Verify Brevo account is active
   - Check backend console for email errors
   - Test with your own email address first

================================================================================
                    TESTING BEST PRACTICES
================================================================================

1. Always test with backend server running
   - Many features require API calls

2. Use browser DevTools (F12)
   - Console tab shows errors
   - Network tab shows API calls
   - Application tab shows localStorage

3. Test on different browsers
   - Chrome (primary)
   - Firefox
   - Edge
   - Safari (if on Mac)

4. Test responsive design
   - Press F12 > Click device icon
   - Test mobile view (375px)
   - Test tablet view (768px)
   - Test desktop view (1920px)

5. Always use TEST Paystack keys locally
   - Never use live keys for testing
   - Test payments are free

6. Check console for errors
   - Red errors mean something's broken
   - Yellow warnings are usually OK

7. Test complete flows
   - Browse → Add to Cart → Checkout → Pay → Track
   - Don't just test individual pages

8. Clear cache when testing CSS changes
   - Ctrl+F5 after CSS modifications
